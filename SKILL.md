# 科技写作之道(Eloquent Science 中文版)

一套用于清晰地写作、修改和展示科学成果的实用体系。

可用于:从零起草、诊断并修复糟糕的草稿、设计图表、准备报告,或评审他人的稿件。
书中的建议适用于任何技术领域——把其中的气象学例子替换成你自己的领域即可。

## 核心理念

科技写作不是文学写作。它的任务是尽可能高效、无歧义地把"理解"传递给读者。
两个词统领一切:

- **简洁(Concision)**——用尽可能少的字说清楚。
- **精确(Precision)**——说出你*确切*想表达的意思,不多不少。

核心比喻是**写作/修改漏斗**:从最大的尺度逐级深入到最小的尺度,
并按同样的顺序进行修改。

1. **文档**——这篇文章有清晰的故事线吗?结构对吗?目标期刊选对了吗?
2. **章节**——每个章节是否各司其职、顺序得当?
3. **段落**——每段是否主旨单一、内部连贯、与前后呼应?
4. **句子**——是否主动、直接、富于变化,没有多余的从句?
5. **词语**——是否简洁、精确,每次都用对了词?
6. **细节**——语法、标点、拼写、格式、参考文献。

在结构稳固之前,绝不要打磨字句。在一个本不该存在的段落里修句子,是白费力气。

## 工作流程

**起草新论文或新章节**
1. 判断这项科研是否值得发表、读者是谁。在动笔*之前*选定目标期刊——它决定了范围、篇幅和风格。
2. 先头脑风暴、列提纲,再起草。先搭出章节骨架,并写下每段一句话的核心主张。
3. 写一份快速、粗糙的初稿。降低门槛:让内心的编辑闭嘴,先写烂,再修。势头比完美更重要。
4. 然后自上而下运行这个漏斗。
→ 各章节必须包含的内容见 `references/paper-structure.md`。

**修改/修复草稿**
1. 按漏斗自上而下通读,先解决大问题,再处理小问题。
2. 把所有章节/小节标题列在一页纸上——单看提纲能否讲清故事?标题是否平行、是否具有描述性?
3. 检查每一段:主旨是否单一、是否连贯(重复、列举、过渡)。
4. 收紧句子:多用主动语态、动作动词,缩短主谓距离。
5. 删除啰嗦短语和软弱词语;确认每个术语都精确。
6. 力求缩短篇幅——收紧后的草稿通常会短 10–20% 且更清晰。
→ 完整的逐遍检查清单见 `references/revision-checklist.md`。

**评审他人稿件**
→ 见 `references/peer-review.md`。

**准备学术报告**
→ 见 `references/presentations.md`。

## 收益最高的习惯

这些习惯能修复绝大多数糟糕的科技写作,应优先应用。

- **一段一个主旨。** 4–8 句为最佳。避免单句成段。段首用主题句点明本段主张。
- **刻意衔接段落。** 连贯来自关键词的*重复*、*列举*(第一、第二、第三)和*过渡*词。读者永远不该疑惑这一段为何接在上一段之后。
- **多用主动语态和动作动词。** "Raleigh 报告了小雪",而非"小雪在 Raleigh 被报告"。删掉软弱动词(是、有、发生、存在、被观测到);选择承载动作的动词。
- **把动作放进动词,把施动者放进主语。** 避免名词化:用"我们比较了",而非"我们进行了比较"。
- **删掉空洞短语。** 用"为了"代替"以便于……的目的";用"因为"代替"由于……这一事实";用"在……附近"代替"在……的邻近区域内"。→ 完整对照表见 `references/word-usage.md`。
- **要精确,不要含糊。** 把"对两个模式的差异进行了考察和讨论"替换为具体的发现。含糊的句子毫无价值。
- **引言要以钩子开头。** 先建立共识,再点出冲突/空白(一个悖论、错误或缺失的知识),然后给出你的回应。没有冲突 = 没有继续读下去的理由。
- **句子长短和节奏要有变化。** 全是短句显得幼稚;全是长句令人疲惫。要交替使用。

## 快速诊断提问

被要求改进一段文字时,运行以下检查并给出具体修改建议:

1. 这一段的唯一主张是什么?如果说不出来,这段就需要拆分或重新聚焦。
2. 哪里用了被动语态?在此处改成主动是否更有力?
3. 哪些词可以删掉而不损失意思?
4. 每个术语都精确吗,还是有含糊/歧义之处?
5. 这一段与上文衔接了吗(重复/过渡)?
6. 朗读一遍——节奏有变化,还是平淡拖沓?
